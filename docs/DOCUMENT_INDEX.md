## 優先度と作成順序

Sprint 2の開発を円滑に進めるため、以下の順序でドキュメントを作成します。

### 優先度: 高（開発開始前に必須）

1. **API設計書** - サーバー/クライアント間のインターフェース契約
2. **データモデル設計書** - 共有型定義の詳細仕様
3. **コンポーネント設計書** - UIコンポーネントの設計指針

### 優先度: 中（開発中に参照）

4. **状態管理設計書** - Zustandストアの詳細設計
5. **データフロー設計書** - イベント処理の流れ
6. **シーケンス図集** - 主要ユースケースの可視化

### 優先度: 低（開発後半〜完了後）

7. **テスト計画書** - テスト戦略の文書化
8. **E2Eテストシナリオ** - テストケースの詳細
9. **デプロイガイド** - 運用ドキュメント
10. **トラブルシューティング** - 運用ドキュメント

## 各ドキュメントの概要

### API設計書 (`API_SPECIFICATION.md`)

WebSocketイベントとRESTエンドポイントの完全な仕様を定義します。

**含まれる内容**:
- WebSocketイベント一覧（イベント名、ペイロード、方向）
- RESTエンドポイント一覧（メソッド、パス、リクエスト/レスポンス）
- エラーコードとハンドリング
- 認証・認可（将来の拡張用）

### データモデル設計書 (`DATA_MODEL.md`)

`src/common/types.ts`で定義する型の詳細仕様を記述します。

**含まれる内容**:
- 各インターフェースの詳細説明
- プロパティの制約（必須/任意、バリデーションルール）
- 型の関係性（ER図的な表現）
- マイグレーション戦略（既存データとの互換性）

### コンポーネント設計書 (`COMPONENT_DESIGN.md`)

UIコンポーネントの階層構造と設計指針を定義します。

**含まれる内容**:
- コンポーネントツリー
- 各コンポーネントのProps定義
- 状態管理の責務分担
- スタイリングガイドライン
- アクセシビリティ要件

### 状態管理設計書 (`STATE_MANAGEMENT.md`)

Zustandストアの詳細設計を記述します。

**含まれる内容**:
- ストアの分割戦略
- 各ストアのstate/actions定義
- セレクターの設計
- 永続化戦略
- デバッグ・DevTools設定

### データフロー設計書 (`DATA_FLOW.md`)

アプリケーション全体のデータの流れを可視化します。

**含まれる内容**:
- progress.md → Server → Client のデータフロー
- ユーザーアクション → Server → ファイル生成のフロー
- エラー伝播の流れ
- キャッシュ戦略

### シーケンス図集 (`SEQUENCE_DIAGRAMS.md`)

主要なユースケースのシーケンス図を集約します。

**含まれる内容**:
- アプリ起動〜ダッシュボード表示
- progress.md更新〜リアルタイム反映
- 指示送信〜タスクファイル生成
- ブロッカー発生〜通知〜解決

### テスト計画書 (`TEST_PLAN.md`)

テスト戦略とカバレッジ目標を定義します。

**含まれる内容**:
- テストピラミッド（Unit/Integration/E2E）
- 各レイヤーのカバレッジ目標
- テストツール・フレームワーク
- CI/CD統合
